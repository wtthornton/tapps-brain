# Experience Events — Payload Schema

TAP-1501 STORY-076.4 / EPIC-076.

`experience_events` is an append-only, monthly-range-partitioned table
(migration `020_experience_events.sql`) that records workflow-level events
produced by agents, MCP tools, and pipeline orchestrators.  Each row is a
single semantic event with optional cross-references to KG objects and private
memories created atomically alongside it.

---

## Overview

| Column | Type | Description |
|---|---|---|
| `id` | `UUID` | Event identifier (auto-generated by the recorder). |
| `tenant_id` / `project_id` | `TEXT` | Tenant scope (RLS-enforced). |
| `brain_id` | `TEXT` | Brain instance identity, e.g. `"tapps-brain"`. |
| `agent_id` | `TEXT` | Agent that produced the event (default `"unknown"`). |
| `session_id` | `TEXT` | Optional — session identifier for grouping. |
| `workflow_run_id` | `TEXT` | Optional — workflow run identifier. |
| `event_type` | `TEXT` | Semantic category (see below). |
| `event_time` | `TIMESTAMPTZ` | Partition key — defaults to `now()`. |
| `subject_key` | `TEXT` | Optional — primary memory key this event relates to. |
| `utility_score` | `REAL` | Measured utility signal `[0, 1]` (default `0.0`). |
| `payload` | `JSONB` | Arbitrary event metadata. |
| `created_memory_key` | `TEXT` | Key of the private memory written by this event. |
| `created_entity_id` | `UUID` | First KG entity upserted by this event. |
| `created_edge_id` | `UUID` | First KG edge upserted by this event. |
| `created_at` | `TIMESTAMPTZ` | Row creation timestamp. |

---

## Supported `event_type` values

The `event_type` column is freeform TEXT with no database constraint.
Well-known types used by the tapps-brain toolchain:

| `event_type` | Trigger |
|---|---|
| `workflow_completed` | A multi-step pipeline or agent loop finished successfully. |
| `tool_called` | An external tool or API was invoked and returned. |
| `approach_failed` | An attempted approach did not succeed; captures failure context. |
| `memory_recalled` | A recall query was executed; captures hit count and utility. |

---

## Atomic write API — ExperienceEventRecorder

All side-effects of one logical event are written in a **single psycopg
transaction**: the `experience_events` row plus optional private memory, KG
entity upserts, KG edge upserts, and evidence inserts.  Any failure on any
component rolls back the entire transaction — including the event row itself.

```python
from tapps_brain.experience import (
    EdgeSpec,
    EntitySpec,
    EvidenceSpec,
    ExperienceEvent,
    ExperienceEventRecorder,
    MemorySpec,
)
from tapps_brain.postgres_connection import PostgresConnectionManager

cm = PostgresConnectionManager(dsn)
recorder = ExperienceEventRecorder(cm, project_id="my-proj", brain_id="tapps-brain")
```

### ExperienceEvent fields

```python
ExperienceEvent(
    event_type="workflow_completed",    # required
    subject_key="recall-result-key",   # optional — related memory key
    utility_score=0.9,                 # float [0, 1]
    payload={"workflow": "plan_and_implement", "steps": 5},
    session_id="sess-abc123",          # optional
    workflow_run_id="wf-def456",       # optional
    memory=MemorySpec(...),            # optional — atomic memory write
    entities=[EntitySpec(...)],        # optional — KG entities to upsert
    edges=[EdgeSpec(...)],             # optional — KG edges to upsert
    evidence=[EvidenceSpec(...)],      # optional — evidence to attach
)
```

### ExperienceResult

```python
result = recorder.record(event)
result.event_id       # UUID of the experience_events row
result.memory_key     # str | None — key of the persisted memory
result.entity_ids     # list[str] — UUIDs in input order
result.edge_ids       # list[str] — UUIDs in input order
result.evidence_ids   # list[str] — UUIDs in input order
```

## Async usage

```python
from tapps_brain.experience import AsyncExperienceEventRecorder

async_recorder = AsyncExperienceEventRecorder(recorder)
result = await async_recorder.record(event)
```

`AsyncExperienceEventRecorder` wraps the sync recorder via `asyncio.to_thread`.
The Postgres I/O runs in a thread-pool thread, keeping the asyncio event loop
unblocked — consistent with `AsyncMemoryStore`.

---

## Partitioning

`experience_events` is partitioned by `RANGE(event_time)` with monthly slices.
Pre-created partitions cover 2026-05 through 2027-04; a default partition
catches rows outside that range.  The recorder inserts without specifying
`event_time` (uses `now()`), so Postgres routes each row to the correct
partition automatically.

---

## RLS enforcement

The table uses FORCE ROW LEVEL SECURITY.  Every connection must set
`app.project_id` before any DML; `ExperienceEventRecorder` does this
automatically via `PostgresConnectionManager.project_context(project_id)`.
Cross-tenant writes fail the `WITH CHECK` clause with a permission error,
rolling back the entire transaction.
